Stages Areas
  • Directorio de Trabajo ➜ Dónde se encuentran los archivos
  • Staging ➜ Área de preparación. El archivo aún no está guardado definitivamente
  • Repositorio Local ➜ Repositorio que se encuentra en nuestro ordenador
  • Repositorio Remoto ➜ Centraliza el repositorio. Varias personas pueden trabajar sobre este.

Estados de Ficheros
  • Untracked ➜ El fichero no está bajo el control de versiones. No está en el repositorio.
  • Unmodified ➜ El fichero está en el repositorio pero no ha sido modificado desde su última versión confirmada.
  • Modified ➜ los datos/ficheros se han modificado, pero todavía no lo has confirmado a tu base de datos.
  • Staged ➜ se ha marcado un archivo modificado en su versión actual para que vaya en tu próxima confirmación.
  • Committed ➜ los datos/ficheros están almacenados de manera segura en tu base de datos local.

Configuración Inicial
  • git --version ➜ Muestra la versión actual de git
  • git config --list ➜ Muestra todas las configuraciones de Git actualmente establecidas por el usuario
  • git config --global user.name "nombre" ➜ Establece el nombre de usuario
  • git config --global user.email "email" ➜ Establece el email del usuario

Comandos Básicos
  • git help "comando" || git "comando" --help ➜ Redirige a una web con información sobre el comando indicado
  • git init ➜ Inicializa un repositorio local
  • git add "fichero" || git add . ➜ Añade un fichero al área de preparación si se especifica, si no, añade todos
  • git commit -m "mensaje" ➜ Registramos los cambios en el repositorio local
  • git push "repositorio" "rama" ➜ Sube los cambios al repositorio remoto

Poniéndonos en marcha
  • git status ➜ Muestra el estado de los ficheros
  • git commit --amend -m "mensaje" ➜ Agrega cambios y modifica el mensaje del último commit
  • git log ➜ Muestra el historial de commits
  • git log --oneline ➜ Muestra el historial de commits en una sola línea
  • git diff || git diff "x""x" ➜ Muestra las diferencias entre el directorio de trabajo y el área de preparación o los commits o ramas especificadas
  • git remote ➜ Muestra los repositorios remotos
  • git remote add "nombre" "url" ➜ Añade un repositorio remoto
  • git remote rename "nombre" "nuevoNombre" ➜ Renombra un repositorio remoto
  • git remote remove "nombre" ➜ Elimina un repositorio remoto
  • git reset HEAD "fichero" ➜ Quita un fichero del área de preparación
  • git branch -M "nombre" ➜ Renombra la rama actual
  • git push -u "repositorio" "rama" ➜ Sube los cambios al repositorio remoto y establece la rama
  • rollback
  • git checkout "rama" ➜ Cambia a la rama indicada
  • git checkout -b "rama" ➜ Crea una rama y cambia a ella
  • git reset --hard "commit" ➜ Elimina los commits posteriores al indicado
  • git reset --soft "commit" ➜ Elimina los commits posteriores al indicado, pero mantiene los cambios
  • git reset --mixed "commit" ➜ Elimina los commits posteriores al indicado, pero mantiene los cambios en el área de preparación
  • git revert "commit" ➜ Crea un nuevo commit que revierte los cambios del commit indicado
  • git revert [nombre_rama]~5 [nombre_rama]~3 ➜ Revierte los cambios de los commits indicados hacia atrás
  • git revert [nombre_rama]~5..[nombre_rama]~3 ➜ Revierte los cambios desde el commit 5 hasta el commit 3 hacia atrás incluyendo ambos
  • etiquetas
  • git tag ➜ Muestra las etiquetas existentes
  • git tag -a "nombre" -m "mensaje" ➜ Crea una etiqueta en el commit actual

Ramas
  • git branch ➜ Devuelve todas las ramas existentes
  • git branch "nombre" ➜ Crea una nueva rama
  • git branch -d "nombre" && git branch -D "nombre" ➜ Elimina la rama especificada tras confirmar con el segundo comando
  • git checkout "rama" ➜ Cambia a la rama indicada
  • git checkout -b "rama" ➜ Crea una rama y cambia a ella
  • git merge "rama" ➜ Fusiona la rama especificada con la rama en la que nos encontramos
  • git merge --abort ➜ Cancela la fusión

Travesía a través de Git

Mi experiencia aprendiendo Git ha sido muy satisfactoria. Al principio, todo me resultaba un poco abstracto, debido a que gran parte del trabajo se hace mediante la terminal de comandos y llevaba mucho tiempo sin trabajar con ellos.

Una vez que entendí los conceptos básicos y empecé a controlar los comandos, comencé a manejarme con mayor facilidad. Me ha gustado mucho descubrir cómo Git facilita el control de versiones, permitiéndo gestionar y colaborar en proyectos de una manera más eficiente.
